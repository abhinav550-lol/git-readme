Project - Git README 

*User can login with their Github account to view their profile information and repositories.
*Based on this information provided by the Github API, we will generate a README file for his/her profile
	-Right of the bat, we will elevate the permission to read user data and repos, but we can also ask for more permissions if we want to add more features in the future
	-Show the preview generated and allow them to download the .md file
	-Or, allow them to directly push the generated README to their Github profile as a new repository named as their username(or something similar)
	-If, it already exists, we need to override the existing README file in the repo with the new one
	-This will include stuff like 
		-Profile picture
		-Bio
		-Top languages used in their repos
		-Badges 
		-Contribution Graph
		-List of their repos with links to them (Project they are working on)
		-Also, for every repo, ask them to ask provide context for the repo (optional)
		-Socials (Optional)
*They can also build project repos - This will require more user interaction for getting the context of the project
